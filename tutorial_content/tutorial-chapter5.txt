        <p><code>acomplete()</code> 可以在 <code>astream()</code> 之上实现：把所有事件 fold 成一个 ProviderResponse。<em>每个具体 provider 只要实现 astream()</em>。</p>

        <CodeBlock filename="src/harness/providers/base.py（accumulate 节选）" lang="python">{`async def accumulate(stream: AsyncIterator[StreamEvent]) -> ProviderResponse:
    """Collect a stream into one ProviderResponse — handles batched tool calls."""
    text_parts, reasoning_parts = [], []
    tool_entries: dict[str, dict] = {}   # id → {"name": str, "args_buffer": str}
    tool_ids_in_order: list[str] = []
    last_opened_id: str | None = None
    orphan_counter = 0
    input_tokens = output_tokens = reasoning_tokens = 0
    reasoning_metadata: dict = {}

    async for event in stream:
        match event:
            case TextDelta(text=t):
                text_parts.append(t)
            case ReasoningDelta(text=t):
                reasoning_parts.append(t)
            case ToolCallStart(id=i, name=n):
                entry_id = i or f"_orphan_{orphan_counter}"
                if not i:
                    orphan_counter += 1
                if entry_id not in tool_entries:
                    tool_entries[entry_id] = {"name": n, "args_buffer": ""}
                    tool_ids_in_order.append(entry_id)
                last_opened_id = entry_id
            case ToolCallDelta(id=i, args_fragment=frag):
                # Fragments reference parent tool_id; fall back to last-opened
                # id; synthesize orphan if a fragment arrives before any Start.
                target_id = i or last_opened_id or f"_orphan_{orphan_counter}"
                if target_id not in tool_entries:
                    tool_entries[target_id] = {"name": "", "args_buffer": ""}
                    tool_ids_in_order.append(target_id)
                tool_entries[target_id]["args_buffer"] += frag
            case Completed(input_tokens=it, output_tokens=ot, ...):
                input_tokens, output_tokens = it, ot
                ...

    # Build tool_calls in arrival order; JSON-parse each args buffer.
    tool_calls = []
    for tid in tool_ids_in_order:
        entry = tool_entries[tid]
        try:
            args = json.loads(entry["args_buffer"]) if entry["args_buffer"] else {}
        except json.JSONDecodeError:
            # Surface raw buffer; registry's validator returns a structured
            # error to the model on the next turn.
            args = {"_raw": entry["args_buffer"]}
        tool_calls.append(ToolCallRef(id=tid, name=entry["name"], args=args))

    if tool_calls:
        return ProviderResponse(tool_calls=tuple(tool_calls), ...)
    return ProviderResponse(text="".join(text_parts), ...)`}</CodeBlock>

        <Callout kind="key" title="什么都不会丢">
          每个 <code>ToolCallStart</code> 开一个按 id 索引的 entry；每个 <code>ToolCallDelta</code> append 到对应 id 的 entry；arrival-order 列表给我们稳定的回放序。<strong>orphan fallback 是防御式的</strong>——一个 fragment 在 start 之前到达的畸形流，会显示为 <code>_orphan_0</code> 而非消失。
        </Callout>

      <Section id="ch5-5" num="5.5" title="Async loop">
        <p>第 4 章的同步 loop 变成 async。逻辑相同，管道不同。</p>

        <CodeBlock filename="src/harness/agent.py" lang="python">{`async def arun(
    provider: Provider,
    registry: ToolRegistry,
    user_message: str,
    transcript: Transcript | None = None,
    system: str | None = None,
    on_event: Callable[[StreamEvent], None] | None = None,
    on_tool_call: Callable[[ToolCall], None] | None = None,
    on_tool_result: Callable[[ToolResult], None] | None = None,
) -> str:
    if transcript is None:
        transcript = Transcript(system=system)
    transcript.append(Message.user_text(user_message))

    for _ in range(MAX_ITERATIONS):
        partial_text: list[str] = []
        try:
            response = await _one_turn(provider, registry, transcript,
                                       partial_text, on_event)
        except asyncio.CancelledError:
            if partial_text:
                transcript.append(Message.assistant_text(
                    "".join(partial_text) + " [interrupted]"))
            raise

        if response.is_final:
            transcript.append(Message.from_assistant_response(response))
            return response.text or ""

        # One assistant message carrying all N ToolCall blocks.
        transcript.append(Message.from_assistant_response(response))

        # Dispatch each call sequentially, in arrival order.
        for ref in response.tool_calls:
            call = ToolCall(id=ref.id, name=ref.name, args=dict(ref.args))
            if on_tool_call:    on_tool_call(call)
            result = registry.dispatch(call.name, call.args, call.id)
            transcript.append(Message.tool_result(result))
            if on_tool_result: on_tool_result(result)

    raise RuntimeError(f"agent did not finish in {MAX_ITERATIONS} iterations")


async def _one_turn(provider, registry, transcript, partial_text, on_event):
    """Run one provider turn; push text deltas into partial_text as we go.
    On CancelledError, accumulated text is still in partial_text — caller flushes it."""
    stream = provider.astream(transcript, registry.schemas())
    async def forward():
        async for event in stream:
            if on_event: on_event(event)
            if isinstance(event, TextDelta):
                partial_text.append(event.text)
            yield event
    return await accumulate(forward())


def run(*args, **kwargs) -> str:
    """Sync wrapper for scripts and tests."""
    return asyncio.run(arun(*args, **kwargs))`}</CodeBlock>

        <Callout kind="key" title="3 个 callback 覆盖一个回合的全部可见表面">
          <ul style={{ margin: 0 }}>
            <li><code>on_event</code>：字符级流式渲染</li>
            <li><code>on_tool_call</code>：宣布调用，显示工具名 + 解析后的参数（不是 JSON-delta 碎片）</li>
            <li><code>on_tool_result</code>：打印结果预览，<strong>✗</strong> 表示 error，<strong>→</strong> 表示成功</li>
          </ul>
          <p>这就是 Claude Code、aider 给你的"agent 在工作、它在做什么"那种感受。</p>
        </Callout>

        <CodeBlock filename="examples/ch05_repl.py（核心）" lang="python">{`async def main():
    transcript = Transcript(system="You are a helpful assistant.")
    last_sigint = 0.0

    while True:
        try:
            user_input = await asyncio.to_thread(input, "\\n> ")
        except EOFError:
            return
        if not user_input.strip(): continue

        task = asyncio.create_task(
            arun(provider, registry, user_input,
                 transcript=transcript, on_event=on_event))

        def on_sigint():
            nonlocal last_sigint
            now = time.monotonic()
            if now - last_sigint < 1.5 and task.done():
                loop.stop()  # 2 次 Ctrl-C 且无任务在跑 → quit
            else:
                last_sigint = now
                if not task.done(): task.cancel()
        loop.add_signal_handler(signal.SIGINT, on_sigint)

        try:
            await task
        except asyncio.CancelledError:

        <CodeBlock filename="src/harness/providers/retry.py" lang="python">{`@dataclass
class RetryPolicy:
    max_attempts: int = 5
    base_delay: float = 1.0
    max_delay: float = 30.0
    max_total_seconds: float = 120.0
    retryable_statuses: frozenset[int] = frozenset({429, 500, 502, 503, 504})

    async def run(self, fn):
        start = asyncio.get_event_loop().time()
        last_exception = None
        for attempt in range(self.max_attempts):
            try:
                return await fn()
            except Exception as e:
                last_exception = e
                if not self._retryable(e):
                    raise
                elapsed = asyncio.get_event_loop().time() - start
                if elapsed >= self.max_total_seconds:
                    raise RetryBudgetExceeded(...) from e
                delay = self._delay(attempt, e)
                await asyncio.sleep(delay)
        raise RetryBudgetExceeded(...) from last_exception

    def _retryable(self, e):
        status = getattr(e, "status_code", None)
        if status is None:
            return isinstance(e, (ConnectionError, TimeoutError,
                                  asyncio.TimeoutError))
        return status in self.retryable_statuses

    def _delay(self, attempt, error):
        retry_after = getattr(error, "retry_after", None)
        if retry_after is not None:
            return float(retry_after)
        jitter = random.uniform(0, self.base_delay)
        return min(self.base_delay * (2 ** attempt) + jitter, self.max_delay)`}</CodeBlock>

        <Callout kind="warn" title="一个值得注意的微妙处：retry 不能 mid-stream">
          如果我们已经向 caller 发出了 3 个 token，从头重试会产生 3 个重复 token。<strong>retry 包<em>打开</em>流；一旦流已经打开，mid-stream 错误就向上冒。</strong>
          第 21 章用 checkpointing 处理 mid-stream 失败：agent 从上一个完成的回合恢复，不是从一个 partial 回合中间。
        </Callout>
      </Section>

      <Section id="ch5-8" num="5.8" title="主备回退 — 当主 provider 倒下">
        <p>当主 provider 长时间不可用，我们想 fallback 到第二个。模式是<strong>一个包裹两个 provider 的 provider</strong>：</p>

        <CodeBlock filename="src/harness/providers/fallback.py" lang="python">{`class FallbackProvider:
    name = "fallback"

    def __init__(self, primary: Provider, secondary: Provider):
        self.primary = primary
        self.secondary = secondary

    async def astream(self, transcript, tools) -> AsyncIterator[StreamEvent]:
        try:
            async for event in self.primary.astream(transcript, tools):
                yield event
            return
        except RetryBudgetExceeded:
            pass   # fall through to secondary
        async for event in self.secondary.astream(transcript, tools):
            yield event

    async def acomplete(self, transcript, tools):
        return await accumulate(self.astream(transcript, tools))`}</CodeBlock>

        <Callout kind="key">
          <p><strong>组合优于继承。</strong> <code>FallbackProvider</code> 自己就是一个 <code>Provider</code>；loop 不知道它是复合的。要三路回退？包两个 <code>FallbackProvider</code>。</p>
          <p>值得标记的注意事项：工具调用的参数形状跨 provider 有细微差异——routed through OpenAI 的同一个工具调用可能产出和 Anthropic 略不同的 JSON。第 22 章显式测试这个；眼下 fallback 对文本响应正确，对工具调用<em>足够形状相近以渡过多数真实失败事件</em>。</p>
        </Callout>
